2. DATA COLLECTED AND COLLECTION METHOD

2.1. Data Stored Locally on Your Device (Not Sent to Any Server)

The following data is stored only on your device in the SwiftData database and is never transmitted to any server:

Data Category Detail
Gold transactions Purchase date, gram amount, amount paid (TRY/USD), jeweler name (optional), notes
Silver transactions Same structure
Financial goals Goal type (gram/TL), target amount, start date, end date
App preferences Notification settings, market data slot selections, language/theme
OCR scanning Receipt photos are discarded after processing; only extracted text is processed transiently

Scope: If your iCloud Backup is enabled, this data is stored within Apple's iCloud infrastructure (subject to Apple's privacy policy). The Developer has no access to this data.

2.2. Data Collected Server-Side (Limited)

Data Purpose Storage Location Retention Period
FCM Push Token Anonymous device identifier issued by Apple/Google to deliver push notifications Firebase Cloud Messaging (Google Ireland Ltd.) + Google Sheets (token only) While the token remains valid or until the user disables notifications
Topic Subscriptions Subscription state for gold_daily, silver_daily, all_users topics (anonymous) Firebase Cloud Messaging While the subscription is active
Threshold Notification Rules Price/percent threshold rules (e.g., "gram_gold > 4500 TL"); no device identifier, only paired with anonymous token Google Sheets (Apps Script) Until the user deletes them
Crash Reports (optional — if collected) Anonymous crash stack traces for debugging; contain no personal data (TBD if added) 90 days

No device advertising identifier (IDFA, IDFV) is collected. No advertising tracking is performed.

2.3. Data Collected by Apple (Developer Has No Access)

During App Store purchases and subscription management, Apple processes the following data under its own privacy policy:

  • Apple ID email (not shared with the Developer)
  • Payment information (not shared with the Developer; only "Premium active/inactive" status)
  • Purchase receipt (for StoreKit verification)
  • Device model, iOS version (App Store Connect analytics, anonymous)

8. DATA SECURITY

8.1. Technical Measures

  • Sandbox and Data Protection mechanisms provided by iOS (Class A: complete file protection);
  • In-app data is stored in iOS Keychain and Encrypted SQLite;
  • Server communication is mandated over HTTPS / TLS 1.2+;
  • FCM token transmission via Firebase's TLS-encrypted channel;
  • Apps Script access logs are active.

8.2. Administrative Measures

  • No one other than the Developer has access to the data;
  • All third-party service providers are subject to their own data processing agreements (Apple DPA, Google DPA).

8.3. Data Breach Notification

In the event of a detected data breach:

  • Notification will be made to the KVKK Authority within 72 hours under KVKK Article 12/5;
  • If EU data subjects are affected, notification will be made to the relevant supervisory authority within 72 hours under GDPR Article 33;
  • Affected users will be notified without delay via in-app notification and/or public announcement.
